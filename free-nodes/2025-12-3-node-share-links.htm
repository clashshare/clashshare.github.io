掌控你的网络命脉：Quantumult神机规则使用全指南

在这个移动网络愈发复杂的时代，互联网自由已经不仅仅是翻墙这么简单，更是一场用户对网络控制权的回归。而在众多iOS平台的代理工具中，Quantumult（俗称“神机”）以其高度的可定制性、精致的界面和强大的规则引擎，成为了众多“进阶玩家”与专业人士的首选。尤其是在规则系统的构建与使用方面，Quantumult不仅赋予了用户前所未有的自由度，更在实战中展现出高效、智能、灵活的网络管理能力。

本文将从入门到进阶，详细讲解Quantumult神机中的规则功能，包括其核心意义、具体配置方法、进阶使用技巧以及常见问题处理。无论你是初识Quantumult的新手，还是想优化已有配置的老用户，都能在本文中找到属于自己的“网络炼金术”。

一、什么是Quantumult神机？

Quantumult是iOS平台上一款功能强大的科学上网工具，其“神机”之名并非浪得虚名。它不仅支持多种代理协议（包括Vmess、Shadowsocks、Trojan等），还提供复杂的分流规则、MITM功能、定制DNS与重写策略，堪称iOS系统中最为精细的网络管理工具之一。

它最大的特点就是：“规则至上”。通过规则，你不仅可以选择哪些流量走代理、哪些直连，还能屏蔽广告、拦截恶意请求、精确流量控制，甚至为不同App制定独立策略，实现真正意义上的“个人网络操作系统”。

二、规则系统的意义：为什么说Quantumult就是一场自定义革命？

Quantumult的规则系统，是其最精华的部分，意义主要体现在以下几个方面：

1. 选择性代理：精准控制访问走向

不再全局代理一刀切，而是：

  • 国外网站走代理；

  • 国内网站走直连；

  • 特定应用（如金融类App）永不代理；

  • 某些服务（如Netflix）使用专线节点。

2. 网络隐私与安全加持

通过规则可屏蔽追踪器、广告域名、已知恶意IP，配合MITM功能甚至可以拦截与重写某些App中的隐藏行为。

3. 节省带宽与优化体验

使用规则将不必要的请求阻断在网络入口，避免带宽浪费。例如广告过滤、无用资源屏蔽等。

4. App定制级分流：一人一网，一网多策

对不同App设置不同的连接方式，比如YouTube走美区节点，TikTok走港区，微信则全部直连，真正实现多线程精细分流。

三、规则类型全解：掌控网络的每一个细节

Quantumult支持多种规则类型，下面是对主流类型的解析：

规则类型示例含义
DOMAINDOMAIN,google.com匹配完整域名
DOMAIN-SUFFIXDOMAIN-SUFFIX,google.com匹配所有以google.com结尾的域名
DOMAIN-KEYWORDDOMAIN-KEYWORD,google匹配所有包含"google"关键字的域名
IP-CIDRIP-CIDR,8.8.8.8/32匹配特定IP地址段
GEOIPGEOIP,CN匹配所有中国大陆IP
FINALFINAL匹配所有未被以上规则命中的流量

每条规则的后面，都会跟一个动作（Action）：

  • DIRECT：直连，不经过代理；

  • PROXY：走默认代理；

  • REJECT：拦截该请求；

  • 选择策略组：如Auto、Netflix、Streaming等。

四、如何设置与导入规则？一步步教你打造专属网络策略

1. 获取规则源

你可以在GitHub上搜索关键词“Quantumult rules”或加入Telegram、微信群等渠道，获取到经过大量用户测试的规则模板。

推荐来源：

  • Loyalsoldier 规则集：https://github.com/Loyalsoldier/v2ray-rules-dat

2. 导入配置文件

Quantumult支持通过多种方式导入配置：

  • 粘贴URL链接；

  • 手动复制粘贴内容；

  • 从iCloud、邮箱、浏览器中导入JSON或txt格式规则文件。

进入Quantumult设置 → 配置 → 导入 → 粘贴规则或添加订阅链接。

3. 编辑规则与策略组

在配置界面点击“规则”→“新建规则”：

  • 类型：选择规则匹配方式（如DOMAIN、GEOIP）；

  • 内容：输入匹配目标，如google.com；

  • 动作：选择直连、代理、拦截或策略组。

例如：

text
DOMAIN-SUFFIX,facebook.com,PROXY
DOMAIN-SUFFIX,baidu.com,DIRECT
GEOIP,CN,DIRECT
FINAL,PROXY

该组合意指：

  • Facebook域名走代理；

  • 百度走直连；

  • 所有中国大陆IP直连；

  • 未匹配部分走代理。

五、策略组管理：让规则更智能、更动态

策略组是Quantumult的核心精髓之一，它允许你将多个节点聚合成一个“智能通道”。

常见策略组类型：

  • 自动测速（Auto）：自动选取最快节点；

  • 手动选择（Manual）：连接前由用户选择节点；

  • 负载均衡（Load Balance）：多节点轮询分流；

  • Fallback：主节点断线自动切备用节点。

使用策略组配合规则，可以实现以下功能：

  • 将Netflix等流媒体流量导向流媒体专用线路；

  • 将Telegram流量导向速度最快的节点；

  • 国内App永远不走代理；

  • 未知请求交由用户手动判断。

六、常见问题解答（FAQ）

Q1：规则导入后无效怎么办？

检查规则位置是否在策略链底部，如果FINAL被提前命中，后面的规则不会生效。规则顺序至关重要，Quantumult自上而下匹配规则，一旦命中即停止匹配。

Q2：如何更新规则？

如果你使用的是带订阅的规则地址，只需设置“定时刷新”即可自动更新。也可手动点击配置页面的“刷新”按钮。

Q3：为何设置了代理规则却还无法访问？

可能原因：

  • DNS未通过代理解析；

  • 节点出现故障；

  • App使用了IP直连，未走规则。
    解决方法：开启“全局DNS代理”或使用MITM功能拦截直连。

Q4：是否可以为某个App单独设置规则？

可以！Quantumult支持“App分流标识”，通过标记进程或端口绑定App，设置专属规则，需iOS 14及以上系统支持。

FAQ

Netch 如何区分全局代理和规则代理？
全局代理会将所有流量通过节点转发，而规则代理仅对指定 IP、端口或应用生效。用户可根据需求选择代理模式，实现流量优化和不同应用的最佳网络体验。
Sing-box 如何实现精确域名分流？
Sing-box 可以按域名设置分流规则，将不同流量发送到指定节点。结合策略组使用，可实现网页、视频和游戏流量精准分配，提高访问速度和网络稳定性。
VMess 使用 TLS+WebSocket 的好处是什么？
TLS+WebSocket 可伪装成正常 HTTPS 流量，增强抗封锁能力和隐蔽性。结合域名伪装和策略组使用，可在高封锁网络环境下保持安全、稳定和高速的网络访问体验。
Shadowrocket 的重写功能有什么用途？
Shadowrocket 的重写功能允许用户在请求阶段修改 HTTP 头、响应内容或重定向 URL。这一功能常用于屏蔽广告、解锁区域限制或调试网页请求，非常灵活强大。
Nekobox 和 Nekoray 有何关系？
Nekobox 是 Nekoray 的移动端衍生版，界面简洁，主要适配 Android 平台。两者使用同样的核心逻辑与配置结构，因此订阅文件和节点信息可以兼容共享。
Shadowrocket 的分流规则如何实现广告拦截？
通过配置规则，将广告域名或请求路径匹配到黑名单策略，使请求直接阻断或重定向到本地地址。这样用户浏览网页或使用应用时可自动屏蔽大部分广告内容。
如何使用 Docker 部署 sing-box 或 v2ray？
选择官方或社区维护的 Docker 镜像，编写 docker-compose.yml 指定配置文件挂载、网络模式与端口映射。使用容器管理可以隔离运行环境、便于日志采集并与反向代理（如 Nginx）协同工作。
Sing-box 如何按 IP 段实现智能分流？
Sing-box 支持根据 IP 段匹配流量，将国内流量直连、国外流量走代理。结合策略组和规则文件，实现精细化流量管理，保证访问速度和稳定性，同时提高节点使用效率。
Trojan 节点无法连接时应如何排查？
遇到 Trojan 无法连接时，需检查服务器证书是否有效、端口是否开放、密码配置是否匹配。还应确认防火墙未阻断相关端口，并尝试切换 TLS 版本或使用 IP 地址连接。
QuantumultX 支持哪些类型的分流规则？
QuantumultX 支持域名、IP、端口、协议和用户自定义脚本分流。结合策略组使用，可实现精准流量控制，如广告屏蔽、视频加速和游戏优化，提高访问速度和体验。
